feedinghabitat

ikamoana.feedinghabitat.feedinghabitat module

This module is implementing the FeedingHabitat class which can simulate the feeding habitat the same way as the SEAPODYM model (2020-08). This class start by initalizing the HabitatDataStructure class using the FeedingHabitatConfigReader module. Then, it performes computation of the Feeding Habitat, for each cohort specified in argument, returned as a DataSet.

class ikamoana.feedinghabitat.feedinghabitat.FeedingHabitat(xml_filepath: str, days_length_float_32: bool = True)

Bases: object

This class can simulate the feeding habitat in the same manner as the SEAPODYM model (2020-08). The FeedingHabitat class first initializes the HabitatDataStructure class using the FeedingHabitatConfigReader module. Then, it performs the feeding habitat calculation for each cohort specified as an argument, returned as a DataSet. It can also calculate the habitat for a cohort that changes over time (moving from one cohort to another).

data_structure

Stores all the data needed to calculate the feeding habitat. Simply print this attribute to get a quick overview of all components.

Type

habitatdatastructure

Examples

First example : Simply initialize with Xml file. Then print some informations about the data used by the module. Finaly, compute Feeding Habitat for the first cohort on complet time, latitude and longitude series.

>>> fh.data_structure.summary()

>>> result = fh.computeFeedingHabitat(cohorts=0)

Second example : Compute Feeding Habitat with sub-part of time, latitude and longitude series for every cohorts.

>>> import numpy as np
>>> fh = ikamoana.feedinghabitat.FeedingHabitat(
...     xml_filepath="./path/to/file.xml")
>>> result = fh.computeFeedingHabitat(
...     cohorts=np.arange(0, fh.data_structure.cohorts_number),
...     time_start=12, time_end=36,
...     lat_min=None, lat_max=40,
...     lon_min=10, lon_max=None,
...     verbose=True)

ikamoana.feedinghabitat.feedinghabitatconfigreader module

This module is used by the feedingHabitat module to setup all variables and parameters. It use a XML configuration file that link to all the netCDF or DYM files used to compute de feeding habitat. It also compute the day length in the area using the PISCES method.

Warning

0.000001 is automatically added on forage layer to copy SEAPODYM behavior.

See also

-

Day length calculation.
